对于吃饭业务的过滤器，先拿碗和先拿筷子都可以，两个一起拿都可以。

1. pojo如何封装？
首先，pojo是啥？百度可知，pojo就是Plain Ordinary Java Object——简单的java对象。
其次，啥叫封装？封装，即隐藏对象的属性和实现细节，仅对外公开接口，控制在程序中属性的读和修改的访问级别；将抽象得到的数据和行为（或功能）相结合，形成一个有机的整体，也就是将数据与操作数据的源代码进行有机的结合，形成“类”，其中数据和函数都是类的成员。（来自百度百科）
所以我认为，封装pojo就是按照pojo的规定和特点乖乖地写java类。希望可以和老师同学们讨论这个问题的详细答案。
2. 复习过滤器如何配置？如何设置通配？
过滤器的配置上文已有不再重复。
如何设置通配？

  • 具体的资源路径：/index.jsp 只有在访问index.jsp这个资源的时候过滤器才会被执行
  • 拦截目录：/user/* 访问/user下的所有资源的时候过滤器才会被执行
  • 后缀名拦截：.jsp 访问后缀名为jsp的资源的时候，过滤器才会被执行
  • 拦截所有资源：/ 访问所有资源，都会被拦截（通配）

4. 登录业务一般要做什么？
（1） 匹配用户名密码，反馈是否登陆成功
（2） 如果登陆成功，检查是否有自动登陆的请求，如果有就把数据封装之后交给cookie；显示登陆成功的页面（跳转到首页）
（3） 如果登陆失败，显示登陆失败的页面（跳转回登陆页）并给出错误信息

5. 退出业务一般要做什么？
（1） 移除session
（2） 创建cookie
（3） 设置cookie过期
（4） 将cookie值返回客户端，使自动登陆信息失效
（5） 重定向到登录页面

6. cookie是什么，一般用于干什么？它在客户端，但服务器端如何控制它？自行搜索并学习cookie的原理。
当一个用户通过 HTTP 协议访问一个服务器的时候，这个服务器会将一些 Key/Value 键值对返回给客户端浏览器，并给这些数据加上一些限制条件，在条件符合时这个用户下次访问这个服务器的时候，数据又被完整地带回给服务器。由于 HTTP 协议是一种无状态协议，当用户的一次访问请求结束后，后端服务器就无法知道下一次来访问的还是不是上次访问的用户，在设计应用程序时，我们很容易想到两次访问是同一人访问与不同的两个人访问对程序设计和性能来说有很大的不同。例如，在一个很短的时间内，如果与用户相关的数据被频繁访问，可以针对这个数据做缓存，这样可以大大提高数据的访问性能。Cookie 的作用正是在此，由于是同一个客户端发出的请求，每次发出的请求都会带有第一次访问时服务端设置的信息，这样服务端就可以根据 Cookie 值来划分访问的用户了。

7. 自动登录的简单原理是什么？
用cookie记录登陆信息，在访问网站时过滤器读取cookie信息并将用户状态上传到session域。

8. 过滤器如何完成自动登录的预先处理动作，再转给Servlet控制?
（1）从请求中获取cookie，似乎能得到很多条信息，所以用一个数组存储
Cookie[] cookies = request.getCookies();
（2）从一堆cookie信息里查找不空的（有登录信息的）cookie，读取cookie信息并上传到session域

9. 第一次就能自动登录吗？
不能，因为第一次登陆的时候没有任何用户信息，所以没法自动登陆。

1. 过滤器和控制器的差别？
（1）配置的方式不同，写法不一样。
（2）作用不同，控制器是执行业务的控制，过滤器执行一些预处理和收尾处理。

2. 过滤器和控制器的url-pattern是什么关系？
是多对多的关系。一个过滤器可以对应多个url-pattern（比如使用通配符的那个）一个控制器的url-pattern可以有多个过滤器。
3. 过滤器配置时，使用通配符在url-pattern中的作用是什么？
<url-pattern>/*</url-pattern> <!-- 对本应用程序的所有请求，均使用此过滤器 -->
通配符能让请求的所有页面都经过这个过滤器

监测的是容器中session对象的创建和sesssion对象的销毁这两个事件。这两个时间分别引起上下文对象中计数器值的增加和减少。

4. 用户会话如何设置过期时间？
（1）在主页面或者公共页面中加入：
session.setMaxInactiveInterval(900);
参数900单位是秒，即在没有活动15分钟后，session将失效。设置为-1将永不关闭。
这里要注意这个session设置的时间是根据服务器来计算的，而不是客户端。所以如果是在调试程序，应该是修改服务器端时间来测试，而不是客户端。
（2）也是比较通用的设置session失效时间的方法，就是在项目的web.xml中设置

这里的15也就是15分钟失效.
（3）直接在应用服务器中设置，如果是tomcat，可以在tomcat目录下conf/web.xml中
找到<session-config>元素，tomcat默认设置是30分钟，只要修改这个值就可以了，负数或0为不限制session失效时间

6. 响应对象如何实现让客户端重定向，再发请求？

7. 对于过滤器和控制器的配置，web.xml方式和注解方式，应该如何实现，应该如何选择？
客户端可能直接访问的url，可以配置在Web.xml里面。客户端不会直接访问的url，可以使用注解配置。
